For this system analysis and usability test, I chose to analyze the Tower Archive from 1922 to 2018 https://cuislandora.wrlc.org/islandora/object/cuislandora%3A67501.

Part 1: System Analysis

Inputs and Outputs The information that goes into an information system is represented by an Input. An input is the direct interaction between the user and the system. Inputs of this system can be represented by the following:

Input Output
Typing search terms into the search bar. Access to archived information in the database (i.e. newspapers).
Mouse clicks to access database options i.e. clicking on the option “The Tower 1981-2000.” Filtering the search results as necessary.
User commands (i.e. downloading a copy of a newspaper from the archive as a PDF file). Downloaded PDF files of copies of archived data.
Software commands (i.e. exporting a citation of the newspaper to an external citation engine). Exported citations to desired destination.

Heuristic Review This is a critique of this website as done using Jakob Nielsen’s 10 Heuristic Principles. This critique is done with the goal in mind of testing the site to see how effective it is in finding attitudes on current events of the era. These archived newspapers act as a window into the time and place that they were published. A student or user could find archives such as this site effective in doing research. Context as such that is provided by these newspapers could be crucial for research purposes and corroborating points made in research papers.

  • Visibility of System Status: The visibility and speed of the system is present and very fast. A student will not have long to wait to get to a desired webpage.
  • Match Between the System and the Real World: Options on the site are spelled out in a plain language with easily understandable terms.
  • User Control and Freedom: When looking at the individual entries on the site, there is highlighted text that links back to where the individual entry is stored on the site and the homepage. If a user needs to go back to the homepage, those options are there. There is, however, an option where the logo of the site is linked to the homepage of JSTOR which is the home of The Tower. Not the homepage of The Tower itself.
  • Consistency and Standards: The designs of the subpages are similar to the homepage so there is a consistent design standard.
  • Error Prevention: If a user tries to download a copy of a newspaper, there is a window that appears confirming that the user has read the terms and conditions of the site before the download is complete. The same warning is given in the window below where someone can create an account.
  • Recognition Rather than Recall: Some of the text in the newspaper can be very hard to read and one may have to use the zoom function in order to properly read the text. This could be due to the nature of what was used to write the text at the time. The text on the site itself is easy to read. Along with this, the search bar function is functionable but not particularly helpful in finding specific information.
  • Flexibility and Efficiency of Use: On the homepage, there are filters on the side of the page where a user can tailor their search. An example of this is the ability to search for newspapers within a date range.
  • Aesthetic and Minimalist Design: The design is simple and straightforward. There is no wasted space or irrelevant information.
  • Help Users Recognize, Diagnose, and Recover from Errors: The Tower itself does not offer a help or support page. JSTOR does this through a link at the bottom of the website.
  • Help and Documentation: The site is easy to navigate and free of complicated systems.

Part 1 Summary The biggest strengths of the system come down to conciseness and consistency. The system is very easy to navigate and straightforward to use at any skill level. Every page has a consistent look and feel.
